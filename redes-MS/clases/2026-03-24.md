# Redes – MikroTik, NAT y Routing

*Fecha: 24/03/2026*

---

## ¿Qué es MikroTik?

MikroTik es un router que integra múltiples funciones de red en un solo dispositivo:

| Función   | Descripción                                      |
|-----------|--------------------------------------------------|
| Routing   | Dirige paquetes entre redes                      |
| Firewall  | Filtra tráfico entrante y saliente               |
| NAT       | Traduce direcciones IP privadas a públicas       |
| WiFi      | Punto de acceso inalámbrico integrado            |

---

## Estructura de red

La topología básica trabajada en clase:
```
Internet
   │
   └── Router MikroTik  (WAN: IP del ISP / LAN: 192.168.88.1)
            │
         Switch
        /   |   \
      PC1  PC2  PC3 ...
```

### Interfaces del router

| Interfaz | Descripción                          | Ejemplo                  |
|----------|--------------------------------------|--------------------------|
| WAN      | Conexión hacia Internet (ISP)        | IP pública o del proveedor |
| LAN      | Red interna (dispositivos propios)   | `192.168.88.0/24`        |

---

## Direccionamiento IP

Ejemplo de configuración utilizada en clase:

| Parámetro       | Valor                          |
|-----------------|--------------------------------|
| IP del router   | `192.168.88.1`                 |
| Red             | `192.168.88.0/24`              |
| Gateway         | `192.168.88.1`                 |
| Rango clientes  | `192.168.88.2 – 192.168.88.254`|

> 💡 La máscara `/24` equivale a `255.255.255.0` y permite hasta 254 hosts en la red.

---

## NAT – Network Address Translation

### ¿Qué problema resuelve?

Las IPs privadas (`192.168.x.x`) **no son enrutables en Internet**. NAT permite que múltiples dispositivos de una red local salgan a Internet compartiendo una sola IP pública.

### Tipos

| Tipo | Traducción          | Uso típico                             |
|------|---------------------|----------------------------------------|
| NAT  | IP → IP             | Una IP privada a una IP pública        |
| PAT  | IP + Puerto → IP + Puerto | Muchos dispositivos, una sola IP pública |

> 📌 El más utilizado en redes domésticas y empresariales es **PAT**, ya que permite múltiples conexiones simultáneas diferenciándolas por puerto.

---

## Firewall

El firewall del MikroTik actúa como filtro entre la red interna y el exterior:

- Permite o bloquea tráfico según reglas configuradas
- Protege la red local de accesos no autorizados
- Opera sobre tráfico entrante (input), saliente (output) y en tránsito (forward)

---

## Routing

El router decide **por dónde enviar cada paquete** según su tabla de rutas.

- Los paquetes destinados a Internet salen por la interfaz WAN
- Los paquetes internos circulan dentro de la LAN sin salir al router
- El **gateway** (`192.168.88.1`) es el punto de salida por defecto para los clientes

---

## Switch

El switch conecta todos los dispositivos dentro de la LAN.

- Opera en **Capa 2** del modelo OSI (usa direcciones MAC)
- No toma decisiones de enrutamiento, solo reenvía tramas dentro de la red local

---

## Flujo completo de un paquete
```
PC cliente
   │
   ▼
Switch  (capa 2, dentro de la LAN)
   │
   ▼
Router MikroTik
   ├── NAT: reemplaza IP privada por IP pública
   ├── Firewall: verifica si el tráfico está permitido
   └── Routing: envía el paquete por la WAN
   │
   ▼
Internet
```

---

## Conceptos clave

| Término  | Definición                                            |
|----------|-------------------------------------------------------|
| LAN      | Red local (dispositivos propios)                      |
| WAN      | Red externa, generalmente Internet                    |
| Gateway  | IP del router, puerta de salida de la red             |
| NAT/PAT  | Traducción de direcciones para salir a Internet       |
| Firewall | Filtro de tráfico para proteger la red                |
| Routing  | Proceso de decidir el camino de cada paquete          |
| Switch   | Dispositivo que conecta equipos dentro de la LAN      |

---

## Resumen

- MikroTik centraliza routing, NAT, firewall y WiFi en un solo equipo
- NAT (específicamente PAT) permite que toda una red privada comparta una IP pública
- El switch opera en capa 2 y no interviene en el enrutamiento
- El flujo de un paquete siempre pasa por: cliente → switch → router → NAT → firewall → Internet
