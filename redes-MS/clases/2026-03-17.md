## Capa 1: Física

> La capa física define el **medio de transmisión**: cables, señales eléctricas, luz y ondas de radio. Es la base de todo el modelo OSI — sin capa física, no hay red.

### Conceptos vistos

| Concepto | Descripción |
|---|---|
| **SNR** (Signal-to-Noise Ratio) | Relación entre la señal útil y el ruido de fondo. Se mide en **dB**. Cuanto mayor el valor, mejor la calidad de la señal. |
| **Interferencias WiFi** | Señales de otros dispositivos (microondas, redes vecinas, Bluetooth) que degradan la comunicación inalámbrica. |
| **Jaula de Faraday** | Estructura metálica que bloquea campos electromagnéticos. Puede aislar completamente señales WiFi y móviles. |
| **Pérdida de luz en fibra óptica** | Atenuación de la señal luminosa al viajar por el cable. Causas: impurezas, curvaturas, empalmes. Se mide en dB/km. |
| **Cable UTP** | *Unshielded Twisted Pair*. Par trenzado sin blindaje, el más común en redes LAN. Categorías típicas: Cat5e, Cat6, Cat6a. |

### Problemas en la capa física

- **Saturación:** La **CPU o la RAM** del dispositivo no pueden procesar más datos. El cuello de botella está en el *host*, no en la red.
- **Congestión:** El **medio de red** no da abasto. Hay más tráfico del que el ancho de banda puede manejar.

> 💡 **Diferencia clave:** Saturación = problema en el dispositivo. Congestión = problema en la red.

## Topología de Red

> Representación de **cómo están conectados** los dispositivos entre sí. Puede ser física (cableado real) o lógica (flujo de datos).

### Herramientas para diagramas de red

- **Microsoft Visio** — estándar en entornos corporativos
- **Draw.io** — gratuito, online, muy usado en la industria

### Elementos comunes en diagramas

- **Macros** — automatizaciones o símbolos reutilizables
- **Modelo** — tipo/referencia del dispositivo (ej: Cisco 2960)
- **MAC Address** — identificador físico único de cada interfaz de red (capa 2)

## Capa 2: Enlace de Datos

> Responsable de la comunicación **nodo a nodo** dentro de una red local. Usa direcciones **MAC** para identificar dispositivos.

### Dispositivo típico: Switch

El **switch** opera en capa 2. Aprende las direcciones MAC de los dispositivos conectados y reenvía tramas solo al puerto de destino correspondiente (a diferencia del hub, que las enviaba a todos).

### Ejemplo de red

```
        INTERNET
            |
       [ Router / DHCP ]
            |
          LAN — 192.168.0.0/24
         /           \
  PCs / WiFi      Otros dispositivos
```

#### Configuración del pool DHCP

El router asigna IPs automáticamente dentro del rango:

```
Inicio:  192.168.0.10
Fin:     192.168.0.100
```

> Los dispositivos fuera del pool (como servidores o impresoras) suelen tener **IP estática** asignada manualmente.

---

## Configuración de red

```
IP:       192.168.0.254
Máscara:  255.255.255.0   →  /24
Gateway:  192.168.0.1
```

### Direcciones reservadas en la red

| Dirección | Rol |
|---|---|
| `192.168.0.0` | Dirección de **red** (identifica la subred, no asignable a hosts) |
| `192.168.0.255` | Dirección de **broadcast** (envía a todos los hosts de la red) |

### Cantidad de hosts posibles en /24

```
2^8 = 256 direcciones totales
256 - 2 = 254 hosts útiles
         ↑ se restan red y broadcast
```

## WiFi

### Bandas de frecuencia

| Banda | SSID / PSK | Características |
|---|---|---|
| **2.4 GHz** | SSID: `ESI-AP13` | Mayor alcance, más interferencias, menor velocidad |
| **5 GHz** | PSK: `ESI-Buceo` | Menor alcance, menos interferencias, mayor velocidad |

> 💡 La banda de **2.4 GHz** atraviesa mejor las paredes pero está más congestionada. La de **5 GHz** es más rápida en espacios abiertos o cercanos al AP.
