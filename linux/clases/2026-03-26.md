## Práctico 1 – Instalación y acceso SSH

**Objetivo:** Instalar Linux, configurarlo en red y acceder remotamente por SSH.

### 1. Configurar IP fija

Editar el archivo de interfaces de red:
```bash
sudo nano /etc/network/interfaces
```

Agregar la siguiente configuración:
```
auto eth0
iface eth0 inet static
  address 192.168.88.200
  netmask 255.255.255.0
  gateway 192.168.88.1
```

Reiniciar el servicio de red para aplicar cambios:
```bash
sudo systemctl restart networking
```

Verificar que la IP quedó asignada:
```bash
ip a
```

### 2. Instalar y habilitar SSH
```bash
sudo apt update
sudo apt install openssh-server -y
sudo systemctl enable ssh
sudo systemctl start ssh
```

Verificar que el servicio está corriendo:
```bash
sudo systemctl status ssh
```

### 3. Gestión de usuarios

Crear un usuario nuevo:
```bash
sudo adduser nombre_usuario
```

Agregarlo al grupo `sudo` para darle permisos de administrador:
```bash
sudo usermod -aG sudo nombre_usuario
```

Verificar a qué grupos pertenece:
```bash
groups nombre_usuario
```

### 4. Conectarse por SSH

Conexión estándar (puerto 22):
```bash
ssh usuario@192.168.88.200
```

Conexión con puerto alternativo (puerto 222):
```bash
ssh usuario@192.168.88.200 -p 222
```

> 📌 El puerto 222 se usa como alternativa al 22 por seguridad o políticas de red.  
> Para cambiarlo, editar `/etc/ssh/sshd_config` y modificar la línea `Port 22`.

## Práctico 2 – Provisión de entorno

**Objetivo:** Preparar la máquina virtual con los servicios necesarios para correr una aplicación.

### Diferencia entre provisión y deploy

| Concepto   | Descripción                                              |
|------------|----------------------------------------------------------|
| Provisión  | Instalar y configurar el entorno (SO, dependencias, servicios) |
| Deploy     | Desplegar el código de la aplicación en ese entorno      |

### Descargar recursos desde URL
```bash
wget https://url-del-recurso
```

### Instalar dependencias según el proyecto

Ejemplo para proyectos Node.js:
```bash
sudo apt install nodejs npm -y
```

Ejemplo para proyectos Python:
```bash
sudo apt install python3 python3-pip -y
```

### Manejo de permisos
```bash
# Ejecutar un comando como administrador
sudo comando

# Dar permisos de ejecución a un script
chmod +x script.sh

# Ejecutar el script
./script.sh
```

## Práctico 3 – Deploy desde GitHub

**Objetivo:** Clonar un repositorio y desplegar la aplicación en la VM.

### 1. Clonar el repositorio

Con HTTPS (sin configuración extra):
```bash
git clone https://github.com/usuario/repositorio.git
```

Con SSH (más seguro, requiere clave):
```bash
git clone git@github.com:usuario/repositorio.git
```

### 2. Generar clave SSH para autenticarse con GitHub
```bash
ssh-keygen -t rsa -b 4096 -C "tu@email.com"
```

Ver la clave pública generada:
```bash
cat ~/.ssh/id_rsa.pub
```

Copiar esa clave y agregarla en GitHub: **Settings → SSH and GPG keys → New SSH key**

### 3. Verificar sesión activa
```bash
who        # muestra usuarios conectados al sistema
whoami     # muestra el usuario actual
```

### 4. Levantar la aplicación

Una vez clonado el proyecto, instalar dependencias y ejecutar:
```bash
cd repositorio
npm install      # o pip install -r requirements.txt, etc.
npm start        # o el comando de inicio del proyecto
```

## Conceptos clave

| Término        | Definición                                                  |
|----------------|-------------------------------------------------------------|
| SSH            | Protocolo para acceso remoto seguro a una terminal          |
| IP fija        | Dirección IP asignada manualmente, no cambia               |
| NAT            | Traduce IPs privadas para salir a Internet                  |
| Bridge         | La VM comparte la red física del host                       |
| Provisión      | Preparar el entorno con todo lo necesario para correr código |
| Deploy         | Desplegar y poner en marcha la aplicación                   |
| Vertical Slice | Entrega funcional de punta a punta de una funcionalidad     |
| Scrum          | Marco ágil de trabajo iterativo por sprints                 |
