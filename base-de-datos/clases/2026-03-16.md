## 📌 ¿Qué es una Base de Datos?

Colección de datos que puede ser utilizada por una o varias aplicaciones.

> Ejemplo: `3.6.1v` → las versiones de una aplicacion

---

## ✅ Características

| Característica | Descripción |
|----------------|-------------|
| **Independencia lógica y física** | Los datos no están ligados directamente al sistema |
| **Sin redundancia** | Cada dato se guarda una sola vez, evitando duplicados |
| **Integridad de datos** | Se aplican reglas y restricciones |
| **Acceso controlado** | Varios usuarios pueden acceder y modificar al mismo tiempo |
| **Respaldo y recuperación** | Backups para proteger la información |

## 🧩 Modelo Entidad–Relación (MER)

Representa la realidad dentro de una base de datos.

### Componentes

| Elemento | Representación |
|----------|---------------|
| **Entidad** | ▭ Rectángulo |
| **Atributo** | ○ Elipse |
| **Relación** | ◇ Rombo |

### Entidades

Representan objetos con características únicas.

Ejemplos: `Transporte`, `Paquete`, `Cliente`, `Empresa`, `Empleado`, `Depósito`

> ⚠️ Las entidades deben tener descripción.  
> Ejemplo: `Nuevo Cliente = Persona nueva`

---

## 🔗 Base de Datos Relacional

```
Persona  ◇  Afiliado
```

### Reglas

- Contempla entidades **fuertes y débiles**.
- Siempre cumplen las propiedades **A.C.I.D.**

| Letra | Significado |
|-------|-------------|
| **A** | Atomicidad |
| **C** | Consistencia |
| **I** | Aislamiento |
| **D** | Durabilidad |

---

## 🔀 Base de Datos No Relacional

**Características:** dinámica, estructura flexible, escala horizontal.

```json
{
  "Nombre": "Documento",
  "Contenido": "...",
  "Usuarios": []
}
```

---

## ⚖️ SQL vs NoSQL

| SQL | NoSQL |
|-----|-------|
| Volumen completo de datos | Sin esquema fijo |
| Necesidad de rapidez | Nuevas versiones rápidamente |
| Sin fallos del sistema | Cambios frecuentes |
