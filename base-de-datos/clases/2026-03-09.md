## 📊 Álgebra Relacional

Concepto fundamental en bases de datos relacionales. Permite realizar operaciones sobre tablas para obtener información.

---

## 🧩 Modelo Entidad-Relación (MER)

El MER se utiliza para diseñar bases de datos.

### Entidades

Representan objetos del mundo real.

Ejemplos: `Profesor`, `Alumno`, `Curso`

### Atributos

Características de una entidad.

```
Profesor
 └── CI (clave primaria)
```

### Relaciones

Conectan entidades entre sí.

```
Profesor → Curso → Alumno
```

---

## 🔢 Cardinalidades

Indican cuántas entidades se relacionan entre sí.

| Tipo  |
|-------|
| 1 : 0 |
| 1 : 1 |
| N : N (M : M) |
| 0 : 1 |
| 0 : 0 |
| 1 : N |

**Ejemplo del cuaderno:**

```
Profesor ─── Curso / Alumno
0:1               1:N
```

---

## 🔑 Claves

### PK — Primary Key (Clave Primaria)

Identifica únicamente cada registro de una tabla.

```
Profesor
CI → Primary Key
```

### FK — Foreign Key (Clave Foránea)

Relaciona una tabla con otra.

---

## 🧱 Entidades Fuertes y Débiles

| Tipo | Descripción | Ejemplo |
|------|-------------|---------|
| **Entidad fuerte** | Existe por sí misma | `Ventas` |
| **Entidad débil** | Depende de otra entidad | `Ticket` |

## 📐 DER / ER — Diagrama Entidad-Relación

Representa gráficamente:

- Entidades
- Relaciones
- Atributos
- Cardinalidades

> Es el paso previo a crear tablas en la base de datos.
