**Tema:** Atributos y Relaciones en el Modelo Entidad-Relación (MER)

## 🧩 Tipos de Atributos

Los atributos describen las propiedades de una **entidad**. Existen cuatro tipos principales:

---

### 🔹 Atributos Atómicos
Son los más simples: **no se pueden dividir** en partes más pequeñas. Son valores únicos e indivisibles.

**Ejemplos:**
- `CI` (Cédula de Identidad)
- `Nombre`
- `Edad`

> 💡 En un diagrama ER se representan con una elipse simple conectada a la entidad.

---

### 🔹 Atributos Multivaluados
Un atributo que puede tener **más de un valor** para una misma entidad.
```
Empleado
 └── Teléfonos
      ├── TL1
      └── TL2
```

📌 Un empleado puede tener varios números de teléfono.

> 💡 En un diagrama ER se representan con una **elipse doble**.

**Otros ejemplos:** correos electrónicos, títulos académicos, idiomas hablados.

---

### 🔹 Atributos Compuestos
Se pueden **dividir en subatributos**, cada uno con significado propio.
```
Dirección
 ├── Nº Puerta
 ├── Calle
 ├── Barrio
 ├── Localidad
 └── Piso
```

📌 "Dirección" no es un dato simple, está compuesto por varias partes independientes.

> 💡 Contraste: `Nombre completo` es compuesto si lo dividimos en `Nombre` + `Apellido`, pero atómico si lo tratamos como una sola cadena.

---

### 🔹 Atributos Derivados
Su valor **se calcula a partir de otros atributos**. No se almacenan directamente, se obtienen en tiempo de consulta.
```
Alumno
 ├── Fecha de nacimiento  ← almacenado
 └── Edad                ← derivado (calculado)
```

📌 `Edad = fecha_actual - fecha_nacimiento`

> 💡 En un diagrama ER se representan con una **elipse punteada**.

**Otros ejemplos:**
- `Total de una factura` → derivado de `precio × cantidad`
- `Antigüedad laboral` → derivado de `fecha de ingreso`

---

## 🔑 Claves (Keys)

Las claves permiten **identificar de forma única** a cada instancia de una entidad o **relacionar** entidades entre sí.

---

### 🔹 Primary Key (PK) — Clave Primaria
Identifica de forma **única e irrepetible** a cada entidad dentro de su tabla.
```
Persona
 └── CI  🔑 (PK)
```

**Reglas:**
- ❌ No puede repetirse
- ❌ No puede ser NULL
- ✅ Debe ser estable (no cambiar con el tiempo)

> 💡 En un diagrama ER el atributo clave se representa **subrayado**.

---

### 🔹 Foreign Key (FK) — Clave Foránea
Es la **PK de otra tabla** que aparece en la tabla actual para establecer una relación entre ellas.
```
Tabla: Ventas
 ├── ID_Venta  🔑 (PK)
 ├── Fecha
 └── ID_Cliente  🔗 (FK → Tabla Clientes)
```

📌 Garantiza la **integridad referencial**: no puede existir un valor FK que no exista como PK en la tabla referenciada.

---

## 🔗 Relaciones

Las relaciones representan las **asociaciones entre dos o más entidades**.

### Estructura general
```
Entidad 1 ──────── Relación ──────── Entidad 2
   PK_1                                 PK_2
```

Las entidades se conectan a través de sus claves.

---

### Cardinalidad

Define **cuántas instancias** de una entidad se relacionan con cuántas de la otra:

| Tipo        | Notación | Ejemplo                        |
|-------------|----------|--------------------------------|
| Uno a Uno   | 1:1      | Persona ↔ Pasaporte            |
| Uno a Muchos| 1:N      | Cliente → Pedidos              |
| Muchos a Muchos | N:M  | Estudiante ↔ Materias          |

---

### Ejemplo práctico
```
Venta ──────── contiene ──────── Tickets
 PK: ID_Venta                    PK: ID_Ticket
                                 FK: ID_Venta
```

📌 Una venta puede estar asociada a uno o varios tickets. El ticket referencia a la venta mediante una FK.

---

## 🧠 Resumen Rápido

| Concepto       | Descripción                          | Símbolo ER          |
|----------------|--------------------------------------|---------------------|
| Atómico        | No se divide                         | Elipse simple       |
| Multivaluado   | Puede tener varios valores           | Elipse doble        |
| Compuesto      | Tiene subatributos                   | Elipse con ramas    |
| Derivado       | Se calcula desde otro atributo       | Elipse punteada     |
| PK             | Identifica únicamente a la entidad   | Atributo subrayado  |
| FK             | Conecta con la PK de otra tabla      | Flecha entre tablas |
