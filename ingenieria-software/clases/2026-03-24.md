# Ingeniería de Software – Introducción

*Fecha: 24/03/2026*

---

## ¿Qué es la Ingeniería de Software?

Disciplina que aplica principios de ingeniería al desarrollo de software, buscando producir sistemas con:

- **Calidad** – que funcione correctamente y sin errores
- **Mantenibilidad** – que sea fácil de modificar en el futuro
- **Eficiencia** – que use bien los recursos disponibles
- **Escalabilidad** – que pueda crecer sin romperse

---

## Características del desarrollo de software

A diferencia de otras ingenierías, el software tiene particularidades propias:

- Es un **producto intangible y complejo** – no se puede tocar ni medir como un objeto físico
- **No se desgasta** físicamente, pero puede volverse obsoleto
- Requiere **planificación y análisis** antes de construirse
- **Depende fuertemente de las personas** – el equipo es el recurso principal
- Evoluciona constantemente junto con los requerimientos
- Tiene **alta complejidad lógica** y depende de una comunicación constante con el cliente

---

## Contexto del negocio

El software no existe en el vacío: se desarrolla dentro de un entorno empresarial con restricciones reales.

| Factor                     | Descripción                                      |
|----------------------------|--------------------------------------------------|
| Estrategia empresarial     | El software debe alinearse con los objetivos del negocio |
| Tiempo de salida al mercado | Llegar tarde puede significar perder ventaja competitiva |
| Costos                     | De desarrollo, mantenimiento y operación         |
| Relación con el cliente    | La comunicación continua es clave                |
| Impacto en el negocio      | Un mal software puede afectar procesos críticos  |

---

## El profesional del software

No alcanza con saber programar. Un buen profesional también necesita:

- Formación técnica sólida
- Capacidad de **trabajo en equipo**
- **Comunicación efectiva** con el cliente y el equipo
- Responsabilidad profesional y ética
- Adaptación al cambio

---

## Fases del desarrollo de software

El desarrollo sigue un proceso con etapas bien definidas:
```
Análisis de requisitos → Diseño → Implementación → Pruebas → Mantenimiento
```

Cada fase alimenta a la siguiente. Saltarse alguna suele generar problemas más adelante.

---

## Modelos de desarrollo

Distintos enfoques para organizar el proceso:

| Modelo              | Característica principal                          |
|---------------------|---------------------------------------------------|
| Cascada             | Fases secuenciales, sin vuelta atrás              |
| Iterativo           | Se construye por versiones sucesivas              |
| Ágil (Scrum, etc.)  | Entregas frecuentes, adaptación continua          |

---

## Factores de éxito en un proyecto

Un proyecto de software se considera exitoso si:

- ✅ Cumple los requerimientos del cliente
- ✅ Se entrega en tiempo y forma
- ✅ Mantiene un nivel de calidad aceptable
- ✅ Es fácil de mantener y escalar

---

## Problemas comunes

| Problema                        | Impacto                                     |
|---------------------------------|---------------------------------------------|
| Cambios en los requisitos       | Retrasos, retrabajo, costos extra           |
| Falta de comunicación           | Funcionalidades incorrectas o incompletas   |
| Errores en el diseño            | Deuda técnica, código difícil de mantener   |
| Dificultad para mantener        | Software que "funciona pero nadie lo toca"  |

---

## Conceptos clave

| Término         | Definición                                              |
|-----------------|---------------------------------------------------------|
| Software        | Producto lógico e intangible                            |
| Ingeniería      | Aplicación de métodos estructurados para resolver problemas |
| Proyecto        | Desarrollo con objetivos, plazos y recursos definidos   |
| Requerimientos  | Necesidades del cliente que el software debe satisfacer |

## Resumen

> El software es complejo y no se puede desarrollar de forma improvisada.  
> La ingeniería de software aporta estructura, métodos y herramientas para hacerlo bien.  
> El éxito de un proyecto depende de tres pilares: **procesos**, **personas** y **herramientas**.  
> La comunicación y los requerimientos son la base de todo.
